关于 Vue-Router

vue-router 简介

通过改变 URL，在不重新请求页面的情况下，更新页面视图。更新视图但不重新请求页面。
先来了解下 vue-router 是如何实现更新页面视图而不请求页面的。

vue-router 中的 mode

mode 有hash、history、abstract 三种模式。默认为 hash 。

  • hash 模式：
    先来理解 url 中的 # 号的含义。

    1. 当浏览器读取 url 为 http://example.com/index.html#user 的时候，会自动将页面滚动到网页标识符为 user 的位置（锚点）。一般可以用 ，或者使用id属性，比如
      来设置锚点。
    2. #只是用来指导浏览器动作，跟 http 请求无关，因此改变 # 后面的内容，是不会发生重新请求资源，加载页面的。
    3. #符号本身以及它后面的字符称之为hash，用 window.location.hash 属性来获取。
    4. 用window.addEventListener(“hashchange”, funcRef, false)、window.onhashchange = func 等方法来监听 hash 发生改变，每次改变会在访问历史中增加一个记录，利用这个特点就能实现更新视图但不重新请求页面。

      通过例子理解 #hash 的使用：
      https://router.vuejs.org/zh-cn/api/options.html#base

  • history 模式

    1. HTML5 history 模式是指 H5 为 history 提供的几个新的方法：

      • history.pushState(data, title, [url]);
      • history.replaceState(data, title, [url])
      • history.state
      • window.onpopstate

        data:存储自定义数据的元素。title：页面标题。url：相对地址，如 index.html。

    2. 显式调用 pushState() 方法，能够将 url 记录到浏览器历史记录栈中。

    3. 通过以上的方市，能够实现添加 url 到浏览器的历史记录中。

      通过例子理解H5 history 的使用：
      http://www.zhangxinxu.com/study/201306/ajax-page-html5-history-api.html?area=pudong

  • abstract 模式
    1. 支持所有 JavaScript 运行环境，如 Node.js 服务器端。如果发现没有浏览器的 API，路由会自动强制进入这个模式。
    2. 简单地理解成是用数组来模拟浏览器历史记录堆栈信息。

hash 跟 H5 history 的区别：
hash 是在当前页面（DOM）改变其下的内容，只是在URL 上生成一个哈希来跟踪状态，而 H5 history 访问的是真实链接，需要后台正确配置。

以上内容，如果想深入理解可以看下 vue-router 的源码：
https://zhuanlan.zhihu.com/p/27588422
https://juejin.im/post/58611353ac502e006739e188

vue-router 简单使用

了解 vue-router 的实现原理后，我们来简单使用下 vue-router 。vue-router 的使用主要分为以下几个步骤:

  1. 引入 vue-router 模块。
  2. 创建 router 对象并配置相关路由和模式。
  3. 在组件里面编写 router 的相关代码。
  4. Vue 对象挂载 router。
router-link

从查看页面元素可以知道，该组件最后会被渲染成带有链接的 a 标签，通过 to 属性指定链接地址。
被选中时会在便签中添加 class 属性值：.router-link-active
router-link 还有很多属性，可以查看官网：

https://router.vuejs.org/zh-cn/api/router-link.html

router-view

该组件主要用来渲染匹配到的路由。

配合 transition：可以给 router-view 设置 transition 过渡（具体用法见Vue2.0 Transition常见用法全解惑）。

配合 使用：可以缓存数据，这样就可以在渲染新的路由组件的时候，不清除上一个路有组件的数据。

关于 router-view 的更多使用方式可以查看官网：

https://router.vuejs.org/zh-cn/api/router-view.html
